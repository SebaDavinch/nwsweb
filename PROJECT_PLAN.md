# NordwindHub — план и состояние проекта

Десктоп-компаньон пилота Nordwind Virtual (Tauri + React, общий backend `server/index.js`).
Документ — снимок плана на 2026-06-15. Детальный журнал работ — в `WORKLOG.md`.

## Архитектура
- **Фронтенд**: React 18 + Vite + Tailwind 4. Приложение живёт на маршруте `/app` (`src/app/components/desktop/`), переиспользует компоненты ЛК (`dashboard/`) и контексты (auth/language/news/notifications).
- **Оболочка**: Tauri (Rust, `src-tauri/`). Frameless-окно, два режима: **Хаб** и **Полёт**.
- **Backend**: `server/index.js` (Express) + модуль `server/email-campaigns.js`. Хранилище — `data/*.json`.
- **В упакованном .exe**: `src/app/api-base.ts` перенаправляет `/api/*` и WebSocket на `https://vnws.org` (CORS на сервере для origin tauri.localhost).
- **Деплой**: единый архив `artifacts/website-release-<ts>.zip` (dist + server + node_modules рантайма + package + deploy/prod + DEPLOY.txt + changelog.json). Без `npm ci` на сервере.

## Прод-заметки (ENV / nginx)
- nginx `/api`: `proxy_http_version 1.1` + `Upgrade`/`Connection "upgrade"` (WebSocket чата).
- `XAI_API_KEY` — ИИ-ассистент. `VITE_DISCORD_APP_ID`/конфиг — Discord RP.
- Email: `EMAIL_PROVIDER`, SMTP_* или EMAIL_API_KEY/URL, `EMAIL_FROM`, `PUBLIC_SITE_URL`, `EMAIL_UNSUB_SECRET`, `EMAIL_LIST_REFRESH_DAYS`, `EMAIL_THROTTLE_MS`. + SPF/DKIM на vnws.org.
- changelog.json кладётся в `data/` (на проде git нет, возим сгенерённый).
- Синхронизация локации пилота: `PILOT_LOCATION_SYNC_INTERVAL_MS` (фон. sweep, дефолт 60 мин), `PILOT_LOCATION_MIN_REFRESH_MS` (лимит форс-обновления на `/api/pilot/location`, дефолт 10 мин), `PILOT_API_LOCATION_SYNC_MS` (синк на `/api/vamsys/me`, дефолт 3 мин). Все три читают живой профиль vAMSYS (forceFresh).
- Куки сессии → `SameSite=None; Secure` для работы входа в упакованном .exe: выставляется env-переменной **`COOKIE_SAMESITE=None`** (по умолчанию `Lax`; при `None` сервер форсит `Secure`, поэтому нужен HTTPS). Включать только на проде с десктоп-сборкой.
